# 《房間戰線》對戰 Helper — 功能規格 v0.1

> 這是給開發用的**方向性規格**（不是實作）。描述這個對戰輔助工具該做什麼、資料結構概念、各流程邏輯。實作細節（確切欄位、每個 transition）等真正開工時再深化。

---

## 0. 一句話定位

**它是《玩家操作 SOP》的可執行版本，外加自己隊伍的機體狀態維護。**

玩家操作自己機體時不必查規則——helper 把「現在能幹嘛、丟幾顆、門檻幾、扣多少、熱多少」都算好、記好。它是**個人的、單機的工具**：你開你的、對手開他的（或不開），互不連線。

### 必讀參考文件（實作前先讀）
- **`_玩家操作SOP_最新.md`** —— **這是 helper 的行為藍本**。helper 的兩個流程入口（輪到你 / 攻擊結算）、近戰三狀況、攻守交握報數，全部嚴格對齊這份 SOP。helper = 把這份 SOP 從「照編號查」變成「跟著點」。
- **`_完整規則_最新.md`** —— 規則本體。所有數值判定（兩段對抗、剋制環、距離有效性、熱、破綻、近戰、姿勢、移動）的權威來源。特別參考：第三章（時間軌/雙系/破綻）、第四章（骰池門檻）、第五章（射擊兩段）、第六章（近戰三狀況）、第七章（移動姿勢）、第八章（熱）、第九章（傷害部位）、附錄B（攻擊流程）、附錄B-2（時間軌記錄圖例＝helper 自動註記的規格）。
- **`_武器表_最新.md`** —— 武器數值（命中/威力/佔格/熱/距離欄/關鍵字）。helper 算骰數門檻的數據源。
- **`_機體卡產生器_最新.html`** —— **衔接來源 + 視覺藍本**。見下「§6 衔接」。helper 讀它的分享字串載入隊伍，且機體卡顯示應與它一致。

## 1. 三條體驗紅線（不可違反）

來自實際使用者（休閒玩家）的要求，這三條是 helper 的設計底線：

1. **骰子留給玩家。** helper 算出「丟 N 顆、門檻 X」，然後**停下來等玩家丟實體骰、輸入過了幾顆**。它是計算機與記憶，不是骰盅。（代骰是**可選開關**，預設關；教學/新手時可開，讓人專注流程。）
2. **沒你事的格自動跳過。** 時間軌推進時，helper 只在「輪到你、且你有機體能決策或有命中格到期」時才停下來找你。收招中、無事的格自動推進，不弹窗煩人。
3. **介面安靜、按鈕大、單手可用。** 平常只顯示機體狀態（熱條、結構條，像遊戲血量）。需要決策時才跳出，問**一個**問題、給**幾個大按鈕**。玩家在客廳單手拿手機、另一手擺模型——單手點幾下就能完成。

## 2. 範圍邊界（做什麼、不做什麼）

### helper 做（全部只關於「我的隊伍」）
- 維護我每台機的狀態：熱、結構、裝甲、武器、掛載點、姿勢、各系當前佔用。
- 推進**我自己的**時間軌視圖，標出我每台機的佔格/命中格/破綻（自動註記，依附錄B-2圖例）。
- 輪到我時，列出**我這格有事的機**（可下指令的、或有命中格到期的）讓我選一台。
- 用人話問情境 → 算骰數門檻 → 自動畫時間軌註記 → 停下等我丟骰輸入 → 更新我的狀態。
- 盯我的熱/結構，快過熱時提醒（變紅）。
- 攻守結算時算**我這端**的數字（我當攻方算命中/威力，我當守方算迴避/結構），與對手口頭報數交握。

### helper 不做
- **不管對手的狀態**（他自己開他的 helper，或用紙筆/心算）。
- **不管主動權、不管「這格輪到誰先」**——那是桌上玩家之間的協調（口頭或那枚硬幣），跨玩家、無連線的工具不該碰。主動權的規則屬於《規則》，不屬於 helper。
- **不連線、不仲裁跨玩家時序**（誰的命中格先結算之類，桌上喊）。
- **不做多人 FFA（拾荒）的全局計分**——個人工具只管自己機體狀態；拾荒的牌/撤離/計分列未來。

> **關於攻守報數**：不論對手有沒有用 helper，流程都一樣（與 SOP 一致）——各自報數。我這端：拿到對手報來的成功數 → 輸進我的 helper → 它幫我比對、算傷害、更新我狀態。對手的數字哪來的（他 helper 算的還是心算）對我無影響。

## 3. 核心資料結構（概念）

### 3.1 機體狀態（每台機一份）
沿用產生器的機體資料（裝甲、結構值、武器清單與掛載點、裝備、標誌武器），**外加對戰時的動態狀態**：
- `currentHeat`（當前熱）／`heatCap`（容量，沿用產生器 `heatCap()`：10 + 散熱器×3）
- `currentStructure`（當前結構）／`maxStructure`（購買值，破壞門檻 = 向下取整半數）
- `posture`（站/蹲/滯空）
- `mountStatus`：各掛載點武器是否報廢（部位破壞用）
- `loaded`：各需裝填武器的當前裝填狀態（已裝幾次/需幾次）

### 3.2 時間軌（multi-array，這是 helper 的引擎核心）
一台機的時間軌 = 一個**二維陣列**：`track[grid][system]`，其中
- `grid` = 第幾格（0..軌道長度）
- `system` ∈ {機動系, 武裝系}
- 每個 cell 記錄該格該系的狀態，至少需表達：
  - `空`（可下新指令）
  - `佔用-命中格`（這格這系是某攻擊的結算點）
  - `佔用-破綻`（佔用但非命中格：起手/收招/裝填裸露）
  - `佔用-全系`（該動作同時吃兩系，如重武器/近戰/高速）
  - `空過`（兩系淨空、散熱4）
  - `移動`（步行/噴射/高速，標距離）
  - `姿勢切換`（附在移動開頭）
- 另記每格的**熱變化**（僅變化時記當前值，對齊附錄B-2「熱不入軌、變化才標」）。

這個 multi-array 就是「自動時間軌註記」的底層：玩家點一個動作，helper 往對應的 grid×system 寫入狀態，前端再依附錄B-2圖例把它畫成 `>`、拉線、框命中格、加粗貫穿、斜槓等視覺。

### 3.3 指針與「有事」判定
- `pointer` = 當前時間格。
- 「我這格**有事的機**」= 該機在這格：有命中格到期要結算（track 上這格是命中格）**或** 至少一系空（可下新指令）。
- helper 推進 pointer 時，逐格檢查我的所有機；無事就自動跳過（紅線2），有事才停下。

## 4. 流程：入口一 —— 輪到你了（對齊 SOP 入口一）

```
pointer 推進到某格：
  掃我所有機 → 列「本格有事的機」
  若無 → 自動推進到下一格（不打擾）
  若有 →
    ① 先顯示三條紅線狀態檢查（熱/結構/門檻），對齊 SOP 步驟1
    ② 列出有事的機，讓玩家【選一台】（多機體選擇——見下）
    ③ 選定一台後，進入該機的「能幹嘛」判定與下指令
```

### 4.1 多機體選擇（隊伍內，不涉對手）
我這格若有多台機可操作（多台能下指令 or 有命中格到期），helper **列出來讓我選一台先操作**。
- **機體是最小決策單位**：選定一台 → 把它這格的事**一次處理完**（結算所有到期命中格 + 下完這格能下的指令），這台本格不再被選。
- 再選下一台，直到我這格所有機都處理完。
- （桌上層面「我跟對手誰先挑、主動權換不換手」由玩家自己協調，helper 不介入；helper 只在「輪到我」這段管好我自己幾台的順序。）

### 4.2 「你現在能幹嘛」判定（殺手功能）
選定一台機後，看它這格兩系狀態（讀 track[pointer]），**亮/灰按鈕**：
- 兩系空 → 全亮：移動／開武器／近戰／裝填／空過
- 只機動系空 → 亮：步行/噴射（不能高速，需全系）；可補武裝攻擊（邊走邊射，標劣勢）
- 只武裝系空 → 亮：開武器/裝填；可補步行/噴射
- 兩系被佔 → 全灰，顯示「這格滿了，照原訂跑」+ 破綻提示
- （邏輯完全對齊 SOP 入口一的情況 3/4/5/6 與「一格最多一機動+一武裝，或一全系」）

### 4.3 下指令 → 自動註記 → 算骰
玩家點一個亮的動作：
- helper 用**人話**問必要細節（不用術語）：開武器→「打哪台？」「他在掩體後嗎？」；移動→「走還是衝？多遠？」；近戰→「砍哪台？」
- 依武器表數值 + 情境，算「丟 N 顆、門檻 X」。
- **自動在 track 寫入佔用**（命中格、破綻、佔系），前端依附錄B-2畫出註記。
- 若是攻擊：標記命中格在未來哪一格；到那格時觸發「入口二」。
- ⚠ 提示：宣告的動作下一格才生效。

## 5. 流程：入口二 —— 攻擊結算（對齊 SOP 入口二，攻守交握）

命中格到期時觸發。helper 算**我這端**的數，與對手口頭報數交握。

### 5.1 我是攻方
對齊 SOP 入口二第1-10步 / 近戰版：
1. helper 顯示「你的命中格到了，XX 對 ?? 用 <武器>」
2. helper 算我的命中骰：基礎命中 + 距離欄（問距離近/遠）+ 情境（目標破綻？被圍？）→「丟 N 顆、門檻 X」
3. 我丟實體骰 → 輸入過了幾顆（或代骰）
4. 我把對手報的迴避成功數輸進來 → helper 比對：我嚴格大於 → 命中，算帶分；否則沒中
5. 命中 → helper 提示「對手吃附熱 +H」（口頭告訴對手）
6. 第二段：問對手裝甲（或他報）→ 查剋制環定門檻 → 算我威力骰(+帶分) → 我丟 → 輸對手結構成功數 → helper 算淨傷（口頭告訴對手扣多少）

### 5.2 我是守方
1. 我輸入對手報來的命中成功數 + 我的情境（有掩體嗎？破綻中嗎？距離對他近/遠？）
2. helper 算我迴避骰：基礎4（+掩體2）+ 破綻劣勢 → 「丟 N 顆、門檻 X」
3. 我丟 → 輸入過幾顆 → helper 比對：對手嚴格大於我 → 中；否則閃掉
4. 中 → 我吃附熱（helper 自動加我的熱）
5. 第二段：helper 用我的裝甲查剋制環定門檻（對手威力 vs 我結構）→ 我丟結構骰 → 輸對手威力成功數 → helper 算我扣多少結構、自動更新
6. helper 收尾檢查：跌破半數→提示部位破壞（骰d6，可在 helper 內骰或實體骰）；歸零→退場

### 5.3 近戰版（對齊 SOP 近戰版三狀況）
helper 依**我（守方）當下武裝系狀態**自動判斷走哪條：
- 我也在命中格 → 近戰值對砍（雙方近戰值）
- 我武裝系空 → 擇優防禦（helper 提示「近戰值 or 迴避值，哪個高用哪個」，**近戰無視掩體：迴避不加2**）
- 我武裝系被佔 or 我無格鬥武器 → 自動命中（helper 跳第一段、無帶分、直接威力）
- 機會攻擊：我若想在被砍的命中格前逃離，helper 警告「會觸發機會攻擊：自動命中、無帶分、剋屬優勢，先結算才脫離」

## 6. 衔接：從產生器讀入隊伍 + 視覺一致

### 6.1 讀入隊伍（複用產生器的編碼）
產生器（`_機體卡產生器_最新.html`）的「複製分享連結」產出格式：
- `encodeState(state)` = `JSON.stringify(state)` → URL-safe base64（`+/=` 換成 `-_`、去尾），載於 `location.hash` 的 `#f=<字串>`。
- helper **直接複用 `decodeState()`** 解這串 → 得到整隊機體資料（裝甲、結構、武器掛載、裝備、標誌武器）。
- 流程：玩家在產生器建好隊伍 → 複製連結 → 貼進 helper → helper 解碼載入 → 開打。免重新輸入。

### 6.2 視覺一致
- helper 顯示機體卡時，**複用產生器渲染機體卡的邏輯**（`statStr()`、`weaponRowHTML()`、掛載點佈局），讓玩家看到的卡與產生器/列印出的卡是**同一張臉**。
- 差別：helper 版在卡上加**動態狀態**——熱條、結構條（會隨對戰變動）、姿勢標記、各武器的裝填/報廢狀態、以及時間軌註記區（依附錄B-2圖例，由 multi-array 自動繪製）。
- 標誌武器照樣顯示命中+1與★（產生器已實作的 signature mod 邏輯）。

## 7. 狀態追蹤與提醒

- **熱條**：顯示 currentHeat/heatCap。接近容量變黃、達容量變紅並提示「過熱：擲骰全劣勢，再增熱即等量扣結構」。helper 在每次加熱後自動檢查、自動套用過熱劣勢到後續骰數計算。
- **結構條**：顯示 currentStructure/maxStructure。跌破半數變色並提示「部位破壞區」，之後每次受創觸發 d6 部位。歸零→該機標記退場、從「有事的機」清單移除。
- **裝填追蹤**：擊發後武器標「空」，需裝填 X 次才能再開；helper 記裝填進度、裝滿才讓該武器可再宣告。
- helper 自動把這些狀態反映到「能幹嘛」判定（過熱→所有骰數標劣勢；武器空→開火按鈕灰掉直到裝填）。

---

## 8. 代骰開關（可選，預設關）

- 預設：helper 只算「丟 N 顆門檻 X」，停下等玩家丟實體骰、輸結果。
- 開啟代骰：helper 替玩家丟（顯示骰面動畫與成功數），玩家確認後繼續。適合教學第一局讓新手專注理解流程。
- d6 部位破壞同理：可實體骰或 helper 內骰。

---

## 9. 實作階段建議

- **階段一（MVP）：1v1**。一台機、跑通核心循環——讀入機體 → 時間軌推進 → 輪到你顯示能幹嘛 → 下指令自動註記 → 攻守結算交握 → 狀態更新。先驗證「SOP 可執行化」這個核心價值。
- **階段二：多機體（2v2/3v3）**。加隊伍內多機體選擇（§4.1）。引擎（multi-array、有事判定）階段一就要為此預留，但 UI 的「選哪台」階段二再做。
- **階段三（未來）**：拾荒 FFA 的牌/撤離（但跨玩家計分仍不做，只記自己手牌與狀態）；產生器與 helper 整合成單一 app。

> 不在範圍：跨玩家連線、主動權仲裁、對手狀態維護。這些永遠留在桌上由玩家協調——helper 始終只是「你這端、操作自己機體時不用查規則」的個人工具。
